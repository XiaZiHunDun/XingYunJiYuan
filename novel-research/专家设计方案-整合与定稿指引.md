## 一、五份文档的共识（可直接写入「定案版」）

以下各专家方向一致，可作为已定策略：

1. **Hard 门禁一票否决**：主角唯一性、里程碑、卷间关键衔接、`review_prompt` 禁忌表述；不通过不得合入。
2. **第二、三卷主路径**：以**分阶段扩写**为主，**局部重写**为辅；先判定骨架/因果，再决定扩写或重写。
3. **流程骨架**：基线评审 → P0 修复 → 扩写试点（POC）→ 规模化扩写/重写 → 全书统稿与终审。
4. **人机分工粗线条**：机器适合字数统计、关键词/正则初筛、格式与术语初检；**Hard 语义判定、S1/S2 因果、情感与节奏**必须人工。
5. **防复发**：维护「雷区/已修复问题」清单 + 批次回归（相邻章 ±3、里程碑章抽审、终盘全量 Hard + 抽样 Soft）。
6. **受众默认**：以网文连载节奏为主，出版适配作为统稿期可选支线或附录指南。
7. **试点必要性**：正式批量前用小样本校准 SOP、工时/千字、Soft 标尺，避免规模化返工。

## 二、关键分歧矩阵（需主编「拍板」写进定案版）

| 议题 | 文档间差异 | 定稿建议（供决策） |
|------|--------------|-------------------|
| **三要素优先级**（体验统一 / 保留 v1 密度 / 工期） | 1：Hard>体验>v1，两害相权可弃 v1 绝对保留；2：工期 P0、可牺牲 v1「绝对保留」但不主动压 v1；4：**可砍工期**拉长到 6–9 月；5：口头「工期不可牺牲」与正文「保风险+体验可延长工期」略有张力 | 建议拆成两句定案：**(A) 质量红线**：Hard > 体验；(B) **资源红线**：在预算内选「日历工期」或「方差目标」，二者冲突时**书面裁示**只砍其一。避免五稿各说各话。 |
| **北极星以字数还是 Soft 为主** | 1：Hard+字数 70%，节拍 20%；2：字数 60% + S1/S2 40%；3：字数 40% + S1/S2 35% + S3–S5 25%，**明确反对**「每 500 字一冲突」作主指标；5：**Soft(S1+S2) 为首要 KPI 60%**，字数次要 | 与 `方法论 v0.4` 对齐：**Hard 非加权**；**Gate0→1→2 顺序**（见 §〇点五）；**不把主观节拍公式写进强制 KPI**（可作扩写自检参考而非合入硬指标）。 |
| **第一卷是否动刀** | 1：可删水（>4500 且重复≥20%），压后 ≥2000 字；2：**绝对不动**为主，删水后 ≥**3000** 字；4：相邻 3 章重复信息删水 | 定案时三选一写死：**冻结 v1 / 仅删水（统一触发条件+删后下限）/ 主动压长对齐**。注意 **2000 vs 3000** 下限冲突必须统一。 |
| **终验「方差」指标** | 1：三卷单章字数方差 ≤**15%**（极严）；3：中位数方差 <**30%** | **15%** 对创作稿往往过严，易倒逼机械凑字。建议改为：**分卷字数中位数差距 ≤X%**（由基线表算一版再定）或仅用「达标率≥90%」+ 主编抽读，避免单一统计指标绑架质量。 |
| **人力与总人天** | 1：8 人、100–130 人天；2：3 人、85–95；3：125–185；4：120–180 | 差异来自**并行度与是否含工具搭建**。定案版应写：**角色最小集合 + 实际人头 + 推导公式**（试点得出「千字扩写人时」再乘待扩存量）。 |
| **篇幅门禁宽度** | 多数 2000–3500；5：**1500–5000** 作批次闸门 | 建议：**常态 2000–3500**；高潮/特例 **主编特批** 并记入 `HISTORY`，不把 5000 字作为默认上限以免结构松散。 |

## 五、从专家意见中应「吸收进 SOP/门禁」的增量（可选增强）

以下不冲突、建议纳入后续 SOP 小版本：
