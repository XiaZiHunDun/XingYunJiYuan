## 〇点五、门禁执行顺序（吸收专家设计共识）

扩写/删水/重写合入前，**固定顺序**，避免「字数达标但逻辑塌了」：

1. **Gate 0 — Hard**：仅通过 / 不通过；见 §2.1。  
2. **Gate 1 — 篇幅**：二三卷默认 **2000–3500 字/章**（口径 §4.4）；高潮等例外须**主编书面特批**并记入 `docs/HISTORY.md`。  
3. **Gate 2 — Soft**：**S1–S7** 见 §2.3；且须满足 **§2.4 扩写基线不退**。  

**不采用**「每固定字数必须含 N 个冲突点」类公式作为**合入硬指标**（可作执行人自检，见 SOP §3.3）。

## 一、核心问题与决策闭合

### 1.1 已识别问题（承接 v0.1）

- 第二、三卷相对第一卷**篇幅过短**（批量重写时被压缩），带来节奏仓促、细节与人物厚度不足。
- 第一卷篇幅更长，存在少量**超长章**，需明确是否处理。

### 1.2 北极星指标（先定这个，再选方案）

**推荐默认（与专家五稿最大公约数一致）**：以 **§〇点五** 的 Gate0→1→2 为周迭代北极星；其中 **Hard 永远先于字数与 Soft**。

任选其一作为**当周 sprint 文字表述**的主标语（勿混用）：

1. **体验统一**：三卷「每章有效信息量」接近（用字数 + Soft 综合，不用机械节拍公式）。  
2. **字数目标**：向网文常见单章字数区间收敛（见 §4.2）。  
3. **风险清零优先**：先保证 Hard 约束与里程碑零事故，再谈扩写。

**v2/v3 扩写风格锚点**：以第一卷**同类型场景**（日常 / 战斗 / 突破等）为参照，做「映射式加厚」——拉长的是**有效信息与情绪**，而非无序堆字（专家设计 2、3 共识）。

### 1.3 方案取舍（回应 v0.1 的「扩 vs 压」未闭合）

| 方案 | 适用条件 |
|------|-----------|
| **A：分阶段扩展 v2/v3（默认推荐）** | 以 v1 为风格与密度参考；v2/v3 **骨架情节正确**，主要缺厚度；不愿大动 v1。 |
| **B：压 v1 + 扩 v2/v3** | 明确要求三卷**行数/字数方差**极小；且接受 v1 **删水**或合并段落带来的润色成本。 |
| **C：维持 v2/v3 现状** | 仅做 Hard 约束修复，不做体验统一。 |
| **D：局部重写** | 当某段**因果链断裂**或**里程碑无法满足**时，扩写无法救，应标为重写范围（与「扩展骨架」边界见 §6）。 |

**默认推荐仍为 A**，理由与 v0.1 一致；若北极星选「方差极小」，应显式切换到 **B** 并记录 v1 可接受删改比例。

### 1.4 v1 超长章（>200 行）策略

在采用方案 A 时：

- **默认**：不强制压缩；仅当评审认定「重复描写 / 同一信息多次交代」时，按 §5.2 做**删水型压缩**。  
- **若切换方案 B**：v1 超长章纳入「删水清单」，**禁止删核心冲突与因果链**。

## 二、约束分层：Hard（硬约束）与 Soft（软指标）

### 2.1 Hard（违反即必须返工）

与 `review_prompt.md` 对齐，包括但不限于：

- **叙事主角唯一性**：全书以林夜 + 苏琳为主线；无不合理的「接班人 / 新主角团 / 代际叙事夺舍」。  
- **禁止类情节**：不出现「传承给新一代当主角」式 handoff；具体措辞排查见 `review_prompt.md` 第三节。  
- **「传承」语义**：禁止的是**叙事主线交接给新一代主角**；世界观内的功法、记忆、物品传承若不与「换主角」绑定，**不自动禁止**——编辑时对照 `review_prompt` 个案判断。  
- **修炼里程碑**：与 `review_prompt.md` 所列章节节点一致；与 `docs/境界进度追踪.md` 冲突时先对表再改文。  
- **卷间关键衔接**：如 ch120→121、ch240→241 等，见 `review_prompt.md`。

Hard **不适用 1–5 分**；结论仅为 **通过 / 不通过**。

### 2.2 Soft（可评分、可排序、可迭代）

| 代码 | 维度 | 说明 |
|------|------|------|
| S1 | 剧情完整性 | 章内目标—障碍—结果是否清楚；卷内因果是否断档 |
| S2 | 逻辑自洽 | 时间线、战力、势力行为是否合理 |
| S3 | 文笔与风格 | 语言、对话、描写是否统一；角色声音区分度；世界观吸引力 |
| S4 | 情感共鸣 | 情绪曲线是否有效（至少一处读者可代入的波动） |
| S5 | 节奏 | 信息投放缓急、战斗/文戏比例是否服务于本章目标 |
| S6 | 可读性/吸引力 | 读者是否愿意一口气读完、是否期待下一章；钩子密度、情绪高点（爽点/笑点/泪点）、代入感、期待感 |
| **S7** | **主角魅力/认同感** | 主角行为是否让读者认同；读者是否想看主角成功；主角是否有独特魅力点 |

**S3 观察项**（不单独打分，记入证据句）：
- 角色声音区分度：每个角色是否有独特的说话方式？对话是否模板化（千人一面）？
- 世界观吸引力：设定是否有新鲜感？读者是否想探索这个世界？

**S4/S5/S6 区分表**（避免重复评分）：

| 维度 | 核心问题 | 判断依据 |
|------|----------|----------|
| **S4 情感共鸣** | 读者情绪是否被调动？ | 是否有泪点/笑点/紧张/感动等情绪反应 |
| **S5 节奏** | 信息投放是否舒服？ | 是否有拖沓/仓促/失衡；信息投放节奏 |
| **S6 可读性** | 读者是否想继续读？ | 章末钩子强度+期待感+点击下一章冲动 |

**简单记忆**：
- S4 = 「你感动了吗？」
- S5 = 「读起来顺不顺？」
- S6 = 「你想继续读吗？」
- **S7 = 「你认同这个主角吗？你想看他成功吗？」**

**S7 主角魅力/认同感评分标准**：

| 分档 | 含义 |
|------|------|
| 5 | 主角魅力极强，读者强烈认同，渴望看到主角成功 |
| 4 | 主角有魅力，读者认同，愿意看主角故事 |
| 3 | 主角基本合格，但缺乏独特魅力点 |
| 2 | 主角行为让读者难以认同 |
| 1 | 主角让读者反感，或毫无魅力 |

### 2.3 软指标量表（建议）

每章（或每个扩写批次）对 **S1–S7** 打 **1–5 分**，并写一句证据（引用原文要点，勿空评）。

| 分档 | 含义 |
|------|------|
| 5 | 达到本书理想样本水平（可对齐 v1 高分章） |
| 4 | 可发，轻微润色即可 |
| 3 | 可发，但扩写/删水优先级中等 |
| 2 | 建议优先处理 |
| 1 | 结构性失败或严重出戏 |

**S6 可读性/吸引力评分标准**：

| 分档 | 含义 |
|------|------|
| 5 | 钩子极强，一口气读完；爽点/笑点/泪点至少2处；读者强烈期待下一章 |
| 4 | 有明确钩子；情绪起伏良好；大部分读者会继续读 |
| 3 | 有基本吸引力；但不突出；读者可能跳章 |
| 2 | 平淡乏味；缺乏钩子；读者可能弃文 |
| 1 | 完全无聊；流水账或自嗨；读者会弃文 |

**批次门槛（建议）**：同一卷内，随机抽样的章 **均分 ≥3.5** 且 **无两项同时 ≤2**；否则暂停扩写，先修风格指南或模板问题。

### 2.4 扩写基线不退（防注水硬规则）

对进入**扩写**流程的章节，在 Gate 2 判定时须同时满足：

- 扩写后 **S1、S2 任一项不得低于扩写前该章基线分数**（若基线已 ≥3.5，则扩写后须 **≥ 基线** 或团队书面接受例外理由）。  
- 若扩写后字数进入区间但 **S1 或 S2 低于基线**：视为 **不合格扩写**，须回退或改扩写策略，**禁止**「凑字数过关」（吸收专家设计 4、5）。

### 2.5 规模化重写占比预警

基线评审完成后，若落在**扩写池之外**、需 **整章级重写** 的章节占 **待处理子集**（通常为 v2+v3 问题章合计）比例 **>20%**，应暂停规模化扩写，由主编评估：是否调整大纲切块、是否切换以重写为主的子阶段（吸收专家设计 3）。**20% 为默认阈值**，可写入 `docs/修订路线图-定案.md` 后调整。

## 三、评审维度与权重（与量表的关系）

| 维度 | 类型 | 权重说明 |
|------|------|----------|
| 主角唯一性及相关禁令 | Hard | 不适用权重，一票否决 |
| 修炼与世界观里程碑 | Hard | 同上 |
| 剧情完整性 | Soft | 高（S1） |
| 逻辑自洽性 | Soft | 高（S2） |
| 文笔质量 | Soft | 中（S3） |
| 情感共鸣 | Soft | 中（S4） |
| 节奏控制 | Soft | 中（S5） |
| 可读性/吸引力 | Soft | 高（S6，与S1/S2并列优先） |

「高/中」用于**排期优先级**：同一阶段内先修 S1/S2/S6，再打磨 S3–S5。

## 四、篇幅：目标、阈值与统计口径

### 4.1 为什么主指标不用「行」

Markdown **行数**受空行、标题、`---`、对话换行习惯影响大；同一信息量可行数差很大。  
**主指标**：建议用 **汉字字数（含标点可按工具定义二选一并固定）** 或「有效叙述段」计数。  
**辅指标**：行数仅作粗筛与历史数据对比。

### 4.2 建议单章字数区间（网文中长篇）

与 v0.1 一致方向：**单章约 2000–3500 字**为舒适区；上下浮动取决于本章是否为高潮章。

### 4.3 扩写 / 压缩触发阈值（可操作）

| 动作 | 建议触发条件（主指标为字数时） |
|------|----------------------------------|
| 考虑扩写 | 明显低于卷目标均值且 **<约 1500 字**（或叙事节拍不足且字数短） |
| 考虑删水压缩 | **>约 4500 字**且评审认定信息重复率高 |
| 仅记录不动作 | 字数在区间内但 Soft 低——走量表与 SOP，而非机械扩/压 |

> v0.1 中「<80 行 / >200 行」可保留为**无字数工具时的粗筛**，但须在报告中注明统计口径。

### 4.4 统计口径（复现用）

团队应固定一种工具规则，例如：

- **字数**：仅统计正文汉字 + 全角标点是否计入，二选一并写死。  
- **行数**：是否包含 `# 章节标题` 行、是否包含仅含空白字符的行。

建议在首次全书统计时，在 `HISTORY.md` 或本目录附录中**记录脚本命令与版本**。

### 4.5 终验与「方差」（勿用单一统计绑架质量）

专家方案中曾出现「三卷字数方差 ≤15%」等极严统计目标，易导致**机械凑字**。

**推荐终验组合**（与 `docs/修订路线图-定案.md` 一致时可照抄）：

1. Gate 1 **达标率**（如二三卷主体章节落入约定区间的比例，默认目标 **≥90%**，余量走特批）；  
2. 分卷字数**中位数**及与基线表对比的**可解释差异**；  
3. 卷末 / 卷首衔接章 **主编抽读**通过。

## 五、优化策略摘要

### 5.1 扩展（丰富厚度）

维度与原则继承 v0.1：**环境、心理、对话、动作、背景**；**扩展≠注水**；**不改变因果与结局**。

额外强调：

- 扩写后易出现的 **模板化排比、说明文式设定灌输** 归入质检（见 SOP）。

### 5.2 压缩（删水）

删除：**重复信息、同义反复、无效寒暄、可合并的动作拆分**。

禁止删：**唯一因果链、里程碑相关对白、伏笔锚点**（对照 `docs/伏笔追踪.md`）。

## 六、扩写 vs 重写的边界

| 情况 | 处理方式 |
|------|----------|
| 骨架事件顺序正确，缺场景与情绪 | **扩写** |
| 视角漂移但未破坏 Hard | 先 **改视角/删旁支** 再扩写 |
| 里程碑无法在不大改前文下成立 | **重写该段 + 回溯修订依赖章** |
| 整段因果断裂、人物动机无法补圆 | **重写**（标 P0） |

## 七、实施优先级（P0–P3）与落点

与 v0.1 一致，但 **P0 必须能落到章节或事件名**：

- **P0**：`review_prompt.md` 所列里程碑章节；已知雷区（如 ch244 长度与跳跃）；专家评审 P0（见 `docs/PROJECT_PLAN.md` 修订说明）。  
- **P1**：各卷末章（ch120、ch240、ch360）及卷首衔接章。  
- **P2**：大战斗/突破章（对照 `docs/境界进度追踪.md` + 大纲）。  
- **P3**：过渡与日常。

具体章节号随大纲迭代调整时，**以 `docs/PROJECT_PLAN.md` 与 `review_prompt.md` 同步更新为准**。

## 九、可读性提升指南（v0.6 新增）

本章节补充「如何让小说更好玩、更吸引读者」的具体指导。

### §九.0 核心工具索引（必读）

方法论工具很多，本节帮助执行者快速找到「现在该用哪个」。

**工具使用场景矩阵**：

| 场景 | 必用工具 | 选 用工具 |
|------|----------|----------|
| **扩写前** | 选择章节模板（9.17）| 信息投放节奏（9.18） |
| **扩写中** | 爽点写作公式（9.15）、章内节奏自检（9.16）、章末钩子自检 | 六类加厚（根据缺什么补什么） |
| **扩写后** | 情感曲线检查（9.6）、记忆点检查（9.13） | 角色弧线追踪（9.12）、卷内节奏检查（9.10） |
| **批次评审** | 弧线检查（9.8）、卷内节奏（9.10） | 信息密度（9.11）、章节分级（9.9） |
| **单章快速自检** | 情感曲线模板（9.6）、记忆点检查（9.13） | 无 |

**简化说明**：
- **必用**：每次扩写都要用，少了会影响质量
- **选 用**：时间充裕时用，或在发现问题时深入检查
- **高级/卷级**：不是单章必做，是复盘时用

**快速定位**：
- 想写爽点？→ 看9.15爽点写作公式
- 不知道章末怎么写？→ 看9.2章末钩子设计
- 章节太平淡？→ 看9.6情感曲线模板
- 不知道这章有没有意义？→ 问「读者会记住什么」（9.13）

### 9.1 爽点设计体系

爽点是网文的核心驱动力。以下是本书适用的爽点类型：

| 爽点类型 | 触发条件 | 本书示例 |
|----------|----------|----------|
| **打脸爽** | 被轻视→当场证明 | 林夜被嘲笑→实力碾压对手 |
| **逆袭爽** | 绝境→翻盘 | 废土崛起→创世境 |
| **羁绊爽** | 被认可/被需要 | 苏琳依赖林夜、并肩作战 |
| **成长爽** | 突破境界 | 突破创世境、实力飞升 |
| **碾压爽** | 实力差距展示 | 对手恐惧、后悔挑衅 |
| **信息差爽** | 读者知道角色不知道 | 暗皇不知道林夜已突破 |

**操作要点**：
- 每5-10章至少安排1个爽点章节
- 爽点前必须有铺垫（被嘲讽→证明），不能无预兆碾压
- 爽点后适当释放情绪，不要突然转入平淡

### 9.2 章末钩子设计

读者会不会点「下一章」，取决于本章结尾是否有吸引力。

**合格钩子类型**：

| 类型 | 示例句式 | 效果 |
|------|----------|------|
| 冲突未解决 | 「但他没想到，暗皇的下一击来得如此之快——」 | 紧张感 |
| 信息未揭示 | 「这个秘密，她从未告诉过任何人——」 | 好奇心 |
| 情绪高点 | 「那一刻，他做出了决定。」 | 期待感 |
| 对比反差 | 「所有人都以为他会逃，但他站了出来。」 | 戏剧感 |
| 意外反转 | 「而真正的问题，不在于暗皇——」 | 惊讶 |

**禁止的章末**：
- 「一天过去了」「众人散去」「回到住处休息」
- 情绪完全平复的收束
- 章节内已完成所有情节，无任何未完事项

### 9.3 情感线设计

林夜+苏琳羁绊是本书核心卖点，必须持续经营：

**情感线节奏**：
- 每10-15章应有1次情感互动（不能全是打怪）
- 甜（日常互动）与虐（误解/牺牲）交替
- 苏琳需要有独立高光时刻，不能只是被保护

**情感描写禁忌**：
- 同一情绪描写重复三遍以上
- 无缘无故的冷漠或疏离
- 羁绊进展过快（情感跳跃需铺垫）

### 9.4 节奏模板（参考）

单章内部建议节奏：

```
开头（200字内）：冲突或悬念出现
├─ 避免：背景介绍、设定铺陈
├─ 推荐：直接进入矛盾

发展（主体）：2-3个小起伏
├─ 每800-1000字有一个小转折
├─ 信息释放与悬念交替

高潮（收尾前200字）：情绪高点或信息释放
├─ 爽点/泪点/重大信息揭露

结尾：留钩子或情绪余韵
├─ 不能是「无事发生」式平收
```

### 9.5 S6「点击下一章」快速判断

评审时额外问一个问题：「如果我是读者，我会点下一章吗？」

| 判断 | S6加分 | 情形 |
|------|--------|------|
| 会 | +1分 | 章末钩子强，强烈想看后续 |
| 可能会 | 不加分 | 有基本吸引力，但不强烈 |
| 不会 | -1分 | 平收、无聊、节奏拖沓 |

### 9.6 情感曲线模板（扩写自检）

每章扩写后自问：本章情感曲线完整吗？

```
本章情感曲线检查：
- 开头（0-200字）：什么情绪？（悬念/压力/好奇/紧张）
- 发展（200-2000字）：情绪如何变化？（期待↑/紧张↑/疑惑/希望）
- 高潮（收尾前200字）：情绪高点是什么？（爽/哭/震惊/愤怒/感动）
- 结尾：是否留钩子让读者点下一章？

合格情感曲线的最低标准：
├─ 有开头情绪（不是平铺直叙）
├─ 有发展起伏（不是一口气平到底）
└─ 有收尾余韵或钩子（不是「无事发生」式结束）
```

**如果缺少任何一个要素，本章是「无效章节」，需要修改。**

### 9.7 锚点章样本库（建立标准）

每种章节类型从v1中选择1章作为S6评分的标准参照。

**锚点章选择标准**：
- 该章读者评价高（无数据时凭专家判断）
- 情感曲线完整（开头/发展/高潮/结尾都有）
- 钩子密度适中，章末留悬念
- 字数在2000-3500区间

**锚点章分类**（待基线评审后填写）：

| 类型 | 锚点章 | S6=5的理由 | 关键特征 |
|------|--------|------------|----------|
| 战斗章 | 待定 | | 爽点明确、章末留悬念 |
| 情感章 | 待定 | | 泪点/羁绊到位、情绪起伏 |
| 日常章 | 待定 | | 有趣、人物性格鲜明 |
| 突破章 | 待定 | | 爽点+成长感、情绪高点 |
| 卷末章 | 待定 | | 高潮+强钩子、期待感极强 |
| 卷首章 | 待定 | | 悬念强、吸引读者追看 |

**使用方式**：
1. 扩写/评审前先读锚点章
2. 对照锚点章评估目标章的钩子密度、情绪高点
3. S6打分时说明「与锚点章XX相比，XXX方面不足/相当」

### 9.8 弧线检查机制（伏笔回收追踪）

单章评分之外，检查伏笔/钩子的**跨章弧线**是否完整。

**新增「伏笔设置-回收追踪」**（在`docs/伏笔追踪.md`中扩展）：

| 伏笔编号 | 设置章节 | 伏笔内容 | 预期回收章节 | 实际回收章节 | 间隔是否合理 |
|----------|----------|----------|--------------|--------------|--------------|
| F001 | ch5 | 「神秘声音」暗示 | ch20 | | |
| F002 | ch10 | 苏琳身世秘密 | ch50 | | |

**弧线检查标准**：
- 伏笔回收间隔：5-30章内为合理（太短=没耐心等，太长=读者忘了）
- 超过30章未回收的伏笔，需在中间章节提醒读者（如角色想起这件事）
- 同一伏笔在不同位置出现两次以上，视为「复发」，检查是否需要合并或删除

**批次评审时增加弧线检查项**：
- 本批设置的伏笔是否记录？
- 本批回收的伏笔是否对应之前设置？
- 伏笔回收间隔是否合理？

### 9.9 章节影响力分级（优化优先级）

不是所有章节都同等重要。按「读者体验影响」分级，优化时优先处理高影响力章节。

| 级别 | 定义 | 特征 | 优化优先级 |
|------|------|------|------------|
| **A级** | 卷末/卷首/高潮章 | 读者期望最高，决定是否继续追读 | 最优先优化（确保S6≥4） |
| **B级** | 里程碑/情感转折章 | 重要事件，情感高点 | 其次（确保S6≥3.5） |
| **C级** | 过渡章/日常章 | 铺垫、节奏调节 | 达标即可（S6≥3） |
| **D级** | 纯铺垫章 | 为A级/B级章服务 | 配合上级章优化，不单独优化 |

**分级方法**（基线评审时完成）：
1. 卷末章（A级）+ 卷首下一章（A级）
2. 高潮章：战斗胜利/突破/情感表白等（A级）
3. 里程碑章：`review_prompt.md`中列出的境界突破点（B级）
4. 情感转折章：关系变化/重要对话等（B级）
5. 日常/过渡章：非以上类型的普通章（C/D级）

**优化资源分配**：
- A级章：最多优化轮次（3轮），主编重点审核
- B级章：标准优化轮次（2轮）
- C/D级章：1轮未达标可接受

### 9.10 卷内节奏检查（每5-10章）

单章节奏之外，还需检查**连续多章的节奏起伏**。每5-10章应形成完整的「小高潮弧线」。

**卷内节奏标准**（每5-10章检查一次）：

| 检查项 | 合格标准 | 不合格表现 |
|--------|----------|------------|
| 小高潮频率 | 每5-10章有1个情绪高点（爽点/泪点/震惊） | 连续10章无情绪高点 |
| 情感变化 | 每5-10章有1次情感转折（希望→绝望/误解→和好） | 情感单调，一直紧张或一直平淡 |
| 悬念设置 | 每5-10章设置1个新悬念或伏笔 | 连续章节无新信息/新悬念 |
| 节奏变化 | 战斗章→日常章→情感章交替 | 连续同类章节超过5章 |

**批次评审时增加卷内节奏检查**：
- 本批10章中，有几个情绪高点？
- 是否有连续章节过于平淡需要插入爽点？
- 卷末章（如ch10, ch20, ch30...）是否为A级，有足够强的情绪高点？

### 9.11 信息密度基线（防注水）简版

扩写不仅要「加字数」，更要「加信息」。以下简化检查，不需要量化统计。

**扩写后快速检查**：

扩写后，本章是否新增了以下**至少一项**？
- [ ] 新冲突（新障碍/新对手/新问题）
- [ ] 新信息（新设定/新角色背景/新地点）
- [ ] 新情感变化（新情绪/关系变化）
- [ ] 新伏笔（新线索/新悬念）

**如果四项都没有，则判定为「注水」，需要重写。**

**简化原因**：
- 「核心事件数」「新信息数」等指标难以量化，不同人统计结果不同
- 用「是否有新增」代替「有多少」，更直观易执行
- 信息密度保障主要靠S1/S2分数（基线不退）和记忆点检查（9.13）

### 9.12 角色弧线追踪

好的小说，主角在每个事件中都有成长或改变。读者应能看到角色的变化轨迹。

**角色弧线检查**（批次评审时）：

| 检查项 | 合格标准 |
|--------|----------|
| 成长点 | 本章是否有角色成长？（学会新技能/突破认知/改变态度） |
| 铺垫 | 这个成长点是否有前期铺垫？（不能突然会了） |
| 影响 | 成长点是否影响后续情节？（不是孤立事件） |

**角色弧线类型**：
- **实力成长**：境界突破/新技能/战斗经验
- **认知成长**：发现真相/理解他人/放下执念
- **情感成长**：接受感情/放下过去/承担责任
- **关系成长**：从陌生→熟悉/从对手→伙伴

**追踪方式**：
- 基线评审时标记「角色弧线节点章」
- 扩写时确保节点章有足够的成长描写
- 批次评审时检查成长点是否被后续章节呼应

### 9.13 记忆点检查（名场面）

读者读完某章后能记住什么？如果什么都不记得，这一章是「无效章节」。

**记忆点类型**：

| 类型 | 例子 | 效果 |
|------|------|------|
| **爆点台词** | "我命由我不由天"、"苏琳，我会回来的" | 读者记住并传播 |
| **震撼场面** | 主角逆转、牺牲、重大发现 | 视觉化记忆 |
| **情感共鸣** | 触动人心的对白/描写 | 情感连接 |
| **悬念设置** | 强烈钩子让读者无法停止 | 追读动力 |

**记忆点检查**（每章扩写后自问）：
1. 本章结束后，读者会记住什么？（一句话总结）
2. 这个记忆点是否有爆点/震撼/共鸣？
3. 还是只是一个「发生过的事情」？

**如果无法回答「读者会记住什么」，本章需要修改。**

**信息收获感**（每章必问）：
- 读者读完本章后，是否感觉「知道了新东西」、「了解了新信息」、「解开了疑惑」？
- 还是感觉「什么都没推进」、「白读了」？
- 每章至少要有**1个信息增量**（新信息/新理解/新悬念解答），让读者有「收获感」
- 信息收获感与记忆点的区别：记忆点侧重「情绪/场面」，信息收获感侧重「认知/情报」

### 9.14 伏笔/钩子链区分

伏笔和钩子不同，但经常被混淆。好的小说两者都有，且节奏不同。

| 概念 | 定义 | 时间跨度 | 节奏 |
|------|------|----------|------|
| **伏笔** | 为后续情节埋线索 | 5-30章 | 慢节奏的信息埋伏 |
| **钩子** | 让读者想继续读 | 每章都有 | 快节奏的情绪拉动 |

**伏笔链**（慢节奏）：
```
设置（ch1）→ 加强（ch5）→ 回收（ch10）→ 新伏笔（ch12）
```
- 伏笔回收间隔：5-30章内合理
- 超过30章需中间提醒（角色想起/相关事件触发）
- 伏笔应与主线相关，不是无关细节

**钩子链**（快节奏）：
```
悬念（ch1结尾）→ 加深（ch2中段）→ 解开（ch2结尾）→ 新悬念（ch2结尾）
```
- 每章必须有至少1个钩子
- 钩子可以是：冲突未解决/信息未揭示/意外反转/情绪高点
- 章末钩子必须强到让读者点下一章

**批次评审时区分检查**：
- 本批设置的伏笔是否记录？预期回收章节？
- 本批的钩子是否有效？章末是否留悬念？
- 伏笔和钩子是否被混淆？（把伏笔当钩子用，或反之）

**伏笔回收满意度**（伏笔回收时自问）：
- 伏笔揭示时，读者是否感到「原来如此！」、「之前埋的果然应验了」？
- 还是感觉「就这？」、「这个揭示太弱了」？
- 伏笔回收时的情绪释放强度应该**不低于埋下时的期待强度**
- 如果回收太晚（超过30章），需要在回收前有「提醒」避免读者忘记

### 9.15 爽点写作公式（实操版）

爽点是网文的核心驱动力。以下是**动作级写作提示**，不是机械的字数占比。

**通用公式：铺垫（动作）→ 压抑（动作）→ 爆发（反应）**

**核心原则**：
- 不要按比例拼积木，会扼杀自然流畅感
- 关注「动作-感受-反应」三环，而非百分比
- 质量比比例重要：铺垫够不够让读者期待？压抑是否真的让人紧张？爆发是否有力？

**打脸爽写法（实操版）**：
```
1. 铺垫：用对手的具体动作/言语建立轻视
   - 示例：「对手冷笑一声，将一块石头丢在林夜面前」
   - 关键：对手行为要让读者感到「这人待会要被打脸」

2. 压抑：写主角的一次实质性受挫，并明确代价
   - 示例：「林夜挡下第三击，嘴角溢出血丝。若再硬接，内腑将受创」
   - 关键：不是简单的「陷入困境」，而是要让读者看到「代价是什么」

3. 转机：通过具体细节变化暗示突破
   - 示例：「林夜眼中的光芒突然变了——那是……」
   - 关键：一个具体的、可感知的变化，不是突然变强

4. 爆发：干脆有力的短句，避免长篇大招
   - 示例：「一拳。光芒炸裂。对手瞳孔骤缩。」
   - 关键：短句有力，情绪释放要快

5. 反应：重点写对手的震惊/后悔，这是爽点释放
   - 示例：「不可能！他的实力怎么会——」
   - 关键：读者想看的是对手的反应，而非主角的pose

6. 钩子：引出新的、更大的问题
```

**逆袭爽写法（实操版）**：
```
1. 铺垫：建立「主角很弱/处于劣势」的认知
   - 用具体场景而非旁白：「他的法器碎了，他的灵力枯了」

2. 压抑：连续2-3次「似乎有希望但又失败」
   - 关键：每次压抑后要有小释放，让读者觉得「快成功了」再压下去

3. 转机：发现关键线索/爆发潜力/外援出现
   - 示例：「他突然想起师父说过的话……」
   - 关键：转机要合理，不能凭空出现

4. 爆发：逆转成功，情绪释放
   - 关键：逆转要干脆，不要拖沓

5. 钩子：新的威胁或更大的挑战
```

**羁绊爽写法（实操版）**：
```
1. 铺垫：日常/并肩，建立关系基础
   - 关键：不要突然表白，用细节暗示感情

2. 危机：一方陷入危险，具体可感知的危险
   - 关键：危险要具体（「她被困在坍塌的矿区」），不要模糊（「她有危险」）

3. 牵挂：另一方的担忧/决心，要有具体动作
   - 示例：「林夜的手在发抖，但他还是冲了过去」
   - 关键：情绪通过动作展现，不是内心独白

4. 回应：羁绊被认可/双向奔赴
   - 关键：要有具体动作或对白，不是空泛的「她感动了」

5. 钩子：关系进一步或新考验
```

**爽点质量检查**：
- 铺垫是否足够？（读者是否真的期待？不是自己觉得期待）
- 压抑是否到位？（情绪是否充分积累？不是压抑一下就爆发）
- 爆发是否有力？（情绪释放是否强烈？反应是否到位？）
- 常见失败：铺垫太长读者失去耐心 / 压抑太久没有小释放 / 爆发太短没感觉 / 忘记写对手反应

**爽点穿透力检查**（写完爽点后自问）：
- 读者读到这里，是否真的激动了？
- 还是感觉「就这？」、「不过如此」？
- 情绪释放是否充分？还是憋着没出来？

**爽点穿透力不足的信号**：
- 铺垫太短，读者还没进入状态就爆发了
- 压抑太轻，读者没有积累足够情绪
- 爆发太快，读者情绪还没释放就结束了
- 缺乏具体细节，只有「他赢了」「他变强了」的概括描述

### 9.16 章内节奏自检表

每章写完后自检，节奏是否紧凑：

**节奏检查项**（每300-500字）：
- [ ] 这300-500字是否有新进展？（新信息/新动作/新对话）
- [ ] 还是只是在重复同一件事？

**情绪检查项**（每800-1000字）：
- [ ] 这800-1000字是否有情绪起伏？（紧张→放松/希望→绝望）
- [ ] 还是一直平铺直叙？

**章末检查**（收尾前200字）：
- [ ] 是否有实质进展？（不是原地踏步）
- [ ] 是否留下钩子让读者点下一章？

**节奏过快/过慢的判断**：
| 问题 | 表现 | 解决 |
|------|------|------|
| 节奏过快 | 事件太多没展开，读者跟不上 | 增加铺垫/压抑/心理描写 |
| 节奏过慢 | 铺垫太长迟迟不推进 | 删减冗余描写，加快节奏 |

### 9.17 类型章节写作模板

不同类型章节有不同的节奏和结构。

**战斗章模板**：
```
结构：敌人嘲讽（铺垫）→ 陷入困境（压抑）→ 转机出现 → 逆转（爆发）→ 章末钩子

开头（200字内）：直接进入战斗或对峙场景
├─ 避免：战斗前的长篇设定介绍
├─ 推荐：敌人先嘲讽，建立打脸预期

发展（主体）：战斗过程
├─ 主角陷入困境（压抑情绪）
├─ 转机出现（增加悬念）
└─ 逆转（爽点爆发）

结尾（200字内）：
├─ 情绪释放（主角胜利）
└─ 章末钩子（新的威胁/下章预告）
```

**情感章模板**：
```
结构：日常互动 → 触发事件 → 误解/冲突 → 和好/表白 → 章末钩子

开头：日常互动（建立关系基础）
├─ 不要：突然表白，太突兀
├─ 推荐：日常中的小细节暗示感情

发展：触发事件打破平衡
├─ 误解/冲突（积累情绪）
└─ 情绪压抑到高点

高潮：和好/表白/情感突破
└─ 要有具体动作/对白，不是空泛描写

结尾：
├─ 情感余韵
└─ 章末钩子（关系进一步/新考验）
```

**日常章模板**：
```
结构：有趣开场 → 小波折 → 信息/伏笔 → 章末钩子

开头（100字内）：有趣/吸引人
├─ 避免：无趣的「他们来到了...」
├─ 推荐：有趣的细节/对话/场景

发展：小波折保持兴趣
├─ 不要：纯粹的流水账
├─ 推荐：日常中的小冲突/趣味

伏笔/信息：可在此处设置
└─ 每章最多1个新设定/伏笔

结尾：钩子
└─ 引导读者期待下一章
```

**突破章模板**：
```
结构：积累（压力/准备）→ 突破（关键时刻）→ 释放（情绪爆发）→ 章末钩子

开头：突破前的积累
├─ 压力/挑战（建立紧迫感）
└─ 准备/铺垫（让突破不突然）

高潮：突破的关键时刻
├─ 具体的感受描写（不要只写「变强了」）
└─ 情绪爆发（成长爽/实力展示）

结尾：
├─ 新的高度（展示突破后的变化）
└─ 章末钩子（新的挑战即将到来）
```

### 9.18 信息投放节奏

信息不能一次性塞给读者，要有节奏地投放。

**每章信息量上限**：
| 类型 | 每章上限 | 原因 |
|------|----------|------|
| 新设定/新世界观 | 1个 | 读者记不住更多 |
| 新角色 | 1-2个 | 需要记忆和认知 |
| 新地点 | 1个 | 场景切换不要太频繁 |
| 关键信息揭示 | 1-2个 | 重要信息需要铺垫和消化 |

**信息揭示节奏**：
```
第一阶段：铺垫（让读者知道「有问题/有秘密」）
├─ 暗示/线索
└─ 字数占比：20-30%

第二阶段：积累（让读者期待）
├─ 更多信息碎片
└─ 字数占比：30-40%

第三阶段：揭示（情绪释放）
├─ 完整揭示
└─ 字数占比：20-30%

第四阶段：消化（让读者接受）
├─ 角色的反应/行动
└─ 字数占比：10-20%
```

**常见问题**：
| 问题 | 表现 | 解决 |
|------|------|------|
| 信息过载 | 一次性揭示太多，读者记不住 | 分批揭示，每章1-2个 |
| 揭示太早 | 读者失去期待 | 增加铺垫和暗示 |
| 揭示太晚 | 读者已经忘记伏笔 | 在30章内回收，超过需提醒 |
| 揭示无铺垫 | 突然出现很突兀 | 先暗示再揭示 |

## 十、评审作业规范（v0.4 增补）

本节规定**「怎么评」**：与 §〇点五「合入评什么」互补；机器产出与人审的分工见下表。

### 9.1 评审类型与产出

| 类型 | 典型输入 | 产出物 | 建议责任人 |
|------|----------|--------|------------|
| **A. 机器全书/批量评审** | 按 `review_prompt.md` 划定的章节范围与上下文 | 结构化评审报告（沿用其模板章节） | 主编指派；**须有人类复核 Hard 与里程碑**，不可直接当定稿 |
| **B. 基线人工校核** | A 的报告 + 原文抽样或全量 P0 候选章 | Hard 确认/驳回；Soft 打分表；缺陷分级清单 | 设定监理 ± 主编 |
| **C. 单章/批次合入门禁** | 当章或 PR 差分 | Gate0→1→2 结论；见 SOP | 扩写执行自检 + 设定监理抽审 |
| **D. 终盘放行评审** | 合并稿或终版分章 + 雷区集 | 签收/退回；残留问题列表 | 主编或独立终审 |

**原则**：机器擅长**穷尽扫描与格式**；人类负责**语义级 Hard、因果与情感判断**。二者结论冲突时，**以人类对 `review_prompt` 的解释为准**。

### 9.2 读窗（上下文范围）

| 场景 | 建议读窗 |
|------|----------|
| 常规单章评审 | 本章 + **相邻各 1 章** |
| 里程碑、卷间衔接、`review_prompt` §七 预警章、P0 | 本章 + **相邻各 3 章**（或至卷首/卷末为止） |
| 纯语言/错别字类 | 可仅本章 |

### 9.3 缺陷分级与处置

| 等级 | 条件（满足任一即归入） | 处置 |
|------|-------------------------|------|
| **H**（Hard 阻断） | `review_prompt` Hard 任一条不通过 | **禁止合入**；回滚、重写或改至通过；记入 `HISTORY.md` 若涉设定变更 |
| **S**（Soft 阻断） | S1 或 S2 **≤2**；或扩写违反 **§2.4 基线不退** | **禁止进入下一扩写批次**；本批次内修毕或回滚 |
| **M**（中优） | S3–S5 任一为 2、或均为 3 | 可合入但进**润色 backlog**；按卷内排期消化 |
| **L**（低优） | 全 Soft ≥4 且无 Hard 疑点 | 通过；仅记录优秀样章备查 |

### 9.4 抽样与覆盖率（默认建议）

团队可按人力上调/下调，**须在基线阶段书面定稿**并记入 `docs/HISTORY.md`：

| 阶段 | Hard | Soft |
|------|------|------|
| **基线** | 对目标范围做**全章**规则扫描（关键词/里程碑表）；对扫描命中与全部已知 P0 候选章做**人工 Hard 确认** | 每卷至少：**全部卷末/卷首衔接候选** + **随机 10 章** + **字数异常章（过短/过长）全量** |
| **扩写批次** | 本批修改章 **100%** Gate0；对每章 **±3** 做衔接速读（可与 SOP 合并） | 本批 **≥30% 随机** 打 **S1–S7**，且**不得少于 5 章**；不足 5 章则全量 |
| **终盘** | **全卷** Hard 复核一次 | **随机 ≥20%** 全 Soft；全部里程碑章建议全量 |

### 9.5 叙事与技法检查清单（辅助，不打分）

评审时可作勾选项，用于写清证据，**映射到 S1–S5** 而非另立维度：

- [ ] **视角**：新增叙述是否主要服务于林夜/苏琳的感知与决策；配角线是否过长夺戏。  
- [ ] **信息链**：本章新设定/新事实是否与**已发表后文**（若已写）冲突；若未写后文，是否与 `docs` 追踪表冲突。  
- [ ] **节奏**：是否存在连续多段「说明设定→说明设定」而无人物反应。  
- [ ] **对话**：是否同时承担推进情节、暴露性格之一；有无车轱辘。  
- [ ] **战斗/动作**：空间关系是否可读；胜负是否依赖已铺垫的能力或道具。

### 9.6 单章评审最小记录（模板）

复制下表即可；**证据**须可定位（原文短语或段落大意，勿空写「不好」）。

```markdown
## ch___ 评审记录
- 读窗：ch___ ~ ch___
- Gate0 Hard：通过 / 不通过 — 证据：
- Gate1 字数：____ 字；区间外特批：有/无（批准人）
- Soft：S1__ S2__ S3__ S4__ S5__ S6__ S7__（各 1–5）
- 缺陷等级：H / S / M / L
- 技法清单异常项：（无则写无）
- 处置：通过 / 回滚 / 扩写 / 重写 / 润色 backlog
- 评审人：____ 日期：____
```
