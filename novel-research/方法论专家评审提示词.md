## 一、评审背景

### 1.1 项目简介

《星陨纪元》是一部玄幻修真+星际科幻小说，360章，三卷本（ch1-360）。目前已完成初稿，存在第二、三卷篇幅不足（平均42-48行/章 vs 第一卷144行/章）的问题，需要建立系统的评审与优化方法论。

### 1.2 待审方法论体系

| 文档 | 用途 | 版本 |

## 二、核心概念速览

### 2.1 门禁体系（Gate 0→1→2）

| 门禁 | 内容 | 通过条件 |
|------|------|----------|
| **Gate 0 - Hard** | 主角唯一性、修炼里程碑、卷间衔接、禁忌表述 | 通过/不通过（一票否决） |
| **Gate 1 - 篇幅** | 单章字数2000-3500 | 达标率≥90% |
| **Gate 2 - Soft** | S1-S7评分（S1剧情完整/S2逻辑自洽/S3文笔/S4情感/S5节奏/S6可读性/S7主角魅力） | 均分≥3.5，S1/S2/S6任一≤2则禁止进入下一批次 |

### 2.2 Soft评分体系（S1-S7）

| 代码 | 维度 | 说明 |
|------|------|------|
| S1 | 剧情完整性 | 章内目标—障碍—结果是否清楚 |
| S2 | 逻辑自洽 | 时间线、战力、势力行为是否合理 |
| S3 | 文笔与风格 | 语言、对话、描写是否统一 |
| S4 | 情感共鸣 | 情绪曲线是否有效 |
| S5 | 节奏 | 信息投放缓急是否服务于目标 |
| S6 | 可读性/吸引力 | 钩子密度、情绪高点（爽点/笑点/泪点）、代入感、是否期待下一章 |
| S7 | 主角魅力/认同感 | 主角行为是否让读者认同；读者是否想看主角成功；主角是否有独特魅力点 |

### 2.3 缺陷分级

| 等级 | 条件 | 处置 |
|------|------|------|
| H | Hard不通过 | 禁止合入 |
| S | S1、S2或S6≤2 | 禁止进入下一批次 |
| M | S3-S5任一为2或均为3 | 可合入，进润色backlog |
| L | 全Soft≥4 | 通过 |

### 2.4 可读性提升指南（§九新增内容）

| 章节 | 内容 |
|------|------|
| 9.1 爽点设计体系 | 打脸爽/逆袭爽/羁绊爽/成长爽/碾压爽/信息差爽 |
| 9.2 章末钩子设计 | 五类合格钩子（冲突未解决/信息未揭示/情绪高点/对比反差/意外反转） |
| 9.3 情感线设计 | 林夜+苏琳羁绊经营（每10-15章1次情感互动） |
| 9.4 节奏模板 | 开头200字冲突、发展2-3个小起伏、高潮、结尾留钩子 |
| 9.5 点击下一章判断 | 快速判断读者是否想继续读 |

### 2.5 扩写策略

- **六类加厚**：环境、心理、对话、动作、背景、爽点、钩子
- **基线不退**：扩写后S1/S2不得低于扩写前

## 三、评审维度

### 3.1 方法论完整性（优先级：高）

- [ ] 是否覆盖了评审全流程（基线→扩写→终验）？
- [ ] Hard/Soft分层是否合理？是否能覆盖主要风险？
- [ ] 门禁顺序（Gate 0→1→2）是否合理？是否有漏洞？

### 3.2 指标科学性（优先级：高）

- [ ] S1-S7评分维度是否合理？是否能反映章节质量？
- [ ] S6「可读性/吸引力」是否能有效判断「好玩/吸引人」？
- [ ] 缺陷分级（H/S/M/L）是否清晰可操作？
- [ ] 批次门槛（均分≥3.5，S1/S2/S6任一≤2禁止进入下一批次）是否合理？

### 3.3 可读性提升指南评审（优先级：高）⚠️重点

- [ ] §九可读性提升指南是否实用、可落地？
- [ ] 爽点设计体系（6种爽点）是否覆盖了网文核心爽感？
- [ ] 章末钩子设计是否具体可操作？
- [ ] 情感线设计是否与本书「林夜+苏琳」核心卖点匹配？
- [ ] 节奏模板是否适用于实际扩写？
- [ ] **情感曲线模板**：是否有可操作的检查项？是否能防止「无效章节」？
- [ ] **锚点章样本库**：是否能建立v1锚点章作为评分标准？
- [ ] **弧线检查机制**：伏笔回收追踪是否能落地？
- [ ] **章节影响力分级**：A-D级是否能指导优化资源分配？

### 3.4 可操作性（优先级：中）

- [ ] SOP是否足够详细可执行？
- [ ] 六类加厚（新增爽点/钩子）是否覆盖了主要扩展需求？
- [ ] 批次回归机制是否完善？
- [ ] 打回上限（2轮）是否合理？

### 3.5 与实践的匹配度（优先级：中）

- [ ] 2000-3500字/章的目标是否符合网文行业实际？
- [ ] 90%达标率是否可实现？
- [ ] 人力配置建议是否合理？

### 3.6 潜在问题识别（优先级：高）

- [ ] 是否有遗漏的边界情况？
- [ ] 是否有过于严格或过于宽松的指标？
- [ ] 专家设计分歧是否已妥善处理？

## 四、具体评审要点

### 4.1 请检查以下关键表述是否存在问题

1. **方法论 §2.1 Hard约束**：是否完整？有无遗漏？
2. **方法论 §2.2 Soft量表（S1-S7）**：1-5分档是否清晰可判断？S6是否与S3-S5有重叠？S7是否与S4/S6重叠？
3. **方法论 §2.4 扩写基线不退**：是否防得住注水？
4. **方法论 §2.5 重写占比预警**：20%阈值是否合理？
5. **SOP §3.1 六类加厚（新增爽点/钩子）**：是否覆盖了所有扩展需求？
6. **SOP §7 雷区回归**：机制是否完善？
7. **§九 可读性提升指南**：是否实用、可落地？

### 4.2 S6可读性/吸引力专项评审 ⚠️重点

1. **S6是否能有效判断「好玩/吸引人」**：
   - 钩子密度的判断标准是否可操作？
   - 爽点/笑点/泪点的定义是否清晰？
   - 「点击下一章」判断是否过于主观？

2. **爽点设计体系是否完整**：
   - 6种爽点（打脸/逆袭/羁绊/成长/碾压/信息差）是否覆盖网文核心爽感？
   - 是否缺少其他常见爽点类型？

3. **与现有S3-S5的边界**：
   - S6与S4（情感共鸣）有何区别？
   - S6与S5（节奏）有何区别？
   - 三个维度是否会重复评分？

### 4.3 请检查以下分歧是否已妥善处理

1. **v1是否动刀**：当前方案默认不动v1，是否合理？
2. **终验方差指标**：当前建议用达标率而非15%方差，是否合理？
3. **篇幅门禁宽度**：2000-3500是否合适？
4. **人天估算**：当前方法论未包含人天估算，是否需要补充？
