# 章节扩展标准操作手册（SOP）

> 版本：1.4（判定树；改视角/删旁支分支；微调分支；增强800字自检）  
> 适用：《星陨纪元》v2/v3 及任意需加厚度的短章  
> 配套：`小说评审优化方法论_v0.4.md`、`../review_prompt.md`

---

## 一、适用范围与目标

- **适用**：单章信息量不足、节拍过密、缺场景与情绪，但**事件顺序与结果**已被采纳为定稿。  
- **不适用**：里程碑不成立、因果断裂——先走「重写」分支（见方法论 v0.4 §6）。

**单章目标（默认）**：扩写后进入 **2000–3500 字** 舒适区（或团队当周书面约定的区间）；同时满足 **§四** 的 Soft 与**基线不退**要求。

**风格锚点**：扩写前在 v1 中任选 **1 章同类型**（日常 / 战斗 / 突破）作参照，记录其信息密度与描写比例，避免无序加长（见方法论 v0.4 §1.2）。

---

## 二、扩写前（必读）

### 2.1 判定树（决定走哪个分支）

```
当前章节
│
├─ Hard门禁是否通过？（主角唯一性/里程碑/代际表述）
│   ├─ 否 → 标记【P0重写】→ 回溯依赖章，重写骨架
│   └─ 是 ↓
│
├─ 骨架是否成立？（事件顺序/因果链/动机是否正确）
│   ├─ 否 → 【局部重写】（保留结果，重写动机/衔接）
│   └─ 是 ↓
│
├─ 配角是否夺戏？（非林夜/苏琳视角超过连续2段）
│   ├─ 是 → 【改视角/删旁支】→ 先删旁支/改视角，再评估
│   └─ 否 ↓
│
├─ 字数是否 <1500 或节拍明显不足？
│   ├─ 是 → 【扩写分支】→ 进入§三（五类/六类加厚）
│   └─ 否 ↓
│
├─ 字数在区间内，但 S3–S5 偏低？
│   ├─ 是 → 【微调分支】→ 不强制扩字，走局部润色
│   └─ 否 → 维持现状，仅Soft评分归档
```

### 2.2 改视角/删旁支（特殊场景）

当配角出场过长（>连续2段）但未破坏Hard时：
1. 先删旁支/改视角（改为林夜或苏琳的有限视角）
2. 评估是否需要补充字数
3. 若需补充，进入扩写分支；若已达标，走微调分支

### 2.3 扩写前准备

1. **读相邻章**：至少精读 **上一章 + 本章 + 下一章**，标出：上章结尾钩子、本章必须承接的信息、下章开头不能撞车的内容。
   - 若本章为 **P0、里程碑、卷间衔接**（见 `review_prompt.md`）：建议扩展阅读至 **±3 章**，避免衔接断层。
2. **Hard 速查**：打开 `../review_prompt.md`，确认本章是否处于**境界/时间线/卷衔接**敏感区。
3. **设定对表**：若涉及境界、势力、道具，对照：
   - `../docs/境界进度追踪.md`
   - `../docs/角色状态追踪.md`
   - `../docs/伏笔追踪.md`（只增不减已埋锚点，除非明确废弃）
4. **列本章骨架**（5 条以内）：目标、障碍、结果、必须出现的设定、禁止出现的内容（如禁代际主角）。
5. **记录扩写基线**：对 S1、S2 各打一次分（或记「未评」），并记当前字数——扩写后须满足方法论 **§2.4 基线不退**。

## 三、扩写中（执行）

### 3.1 六类加厚（按缺什么补什么，非六类全上）

| 类型 | 补什么 | 禁止 |
|------|--------|------|
| 环境 | 可感知的声光气味、空间关系、压迫感 | 纯景物堆砌无人物反应 |
| 心理 | 决策代价、恐惧与欲望、与苏琳/羁绊相关情绪波动 | 同一情绪段重复三遍以上 |
| 对话 | 推进信息、暴露性格、制造误解/默契 | 车轱辘话、说明书式世界观 |
| 动作 | 战斗/移动的空间逻辑、胜负关键一击的来由 | 无后果的「为打而打」延长 |
| 背景 | 只补**本章决策需要**的势力/历史切片 | 大段设定朗诵 |
| **爽点** | 打脸/逆袭/羁绊确认/实力展示后的情绪释放段落 | 无铺垫的突然碾压 |
| **钩子** | 结尾留悬念、冲突未解决、信息未揭示 | 平收（"一天过去了""众人散去"式） |

### 3.1.1 提升 S6 可读性/吸引力的扩写技巧

**钩子密度**：每章应有 1-2 个「读者想知道接下来」的点
- 章节开头：冲突或悬念在前 200 字内出现
- 章节结尾：留下未解之谜、对比落差或情绪高点
- 章中：每 800-1000 字有一个小转折或信息释放

**情绪高点设计**（爽点/笑点/泪点）：
- **爽点**：主角逆转、被打脸方当场后悔、实力压制展示、羁绊被认可
- **泪点**：牺牲、离别、误解后和解、回忆杀（慎用）
- **笑点**：意外反转、人物反差萌、吐槽

**代入感增强**：
- 决策时写清代价（不只是结果，还有"如果失败会怎样"）
- 写恐惧与欲望（主角真正想要什么、害怕什么）
- 与苏琳/羁绊相关的情绪波动（见方法论 v0.4）

**禁忌**：
- 流水账式平铺直叙（无起伏）
- 连续三段以上纯说明无人物反应
- 章节内塞入过多新设定（每章≤1个新信息块）

### 3.1.2 章末钩子设计规范

**好的章末钩子类型**：

| 类型 | 例子 | 效果 |
|------|------|------|
| 冲突未解决 | 「但他没想到，暗皇的下一击来得如此之快——」 | 读者紧张，想知道结果 |
| 信息未揭示 | 「这个秘密，她从未告诉过任何人——」 | 读者好奇，想知道是什么 |
| 情绪高点 | 「那一刻，他做出了决定。」 | 读者期待后续 |
| 对比反差 | 「所有人都以为他会逃，但他站了出来。」 | 读者想看后续 |
| 意外反转 | 「而真正的问题，不在于暗皇——」 | 读者惊讶，想知道真相 |

**禁止的章末**：
- 「一天过去了」「众人散去」「回到住处休息」
- 情绪完全平复的收束（无余韵）
- 章节内已完成所有情节，无任何未完事项

**检查点**：扩写完成后自问「读者会想点下一章吗？」——如果答案是否，修改章末。

### 3.2 视角与主角线

- 新增段落默认 **林夜或苏琳的有限视角**；若必须写他人反应，控制在 **服务主线感知** 的长度内。  
- 不写「下一代将继承一切」式收束；措辞排查以 `review_prompt.md` 为准。

### 3.3 扩写过程中的自检（每写约 800 字停一次）

- [ ] 是否出现 **同一比喻结构** 连续两次以上？（删一改一）
- [ ] 是否出现 **设定列表**（连续三段无人物）？（改为对话或动作带出）
- [ ] 是否 **新增与前后章矛盾** 的事实？（立即修正或标注待对表）
- [ ] **章末钩子自检**：本章结尾是否是「一天过去了」「众人散去」式平收？是否留有悬念/冲突/未解问题？
- [ ] **爽点检查**：本章是否有情绪高点（爽点/泪点/笑点）？如果还没有，考虑在下一段加入
- [ ] **视角检查**：新增段落是否以林夜/苏琳视角为主？配角是否夺戏？

### 3.4 微调分支（不强制扩字）

**适用场景**：字数已在区间（2000-3500），但 S3-S5 偏低

**操作方式**：
1. 不增加新情节或大幅扩写
2. 局部润色：删模板化排比、调对话信息密度、收束节奏
3. 检查是否有情绪高点可增强（爽点/泪点/笑点）
4. 仍须过 Gate 0（Hard门禁）

## 四、扩写后（质检门禁）

### 4.1 Hard 门禁（不通过则不得合入）

按 `../review_prompt.md` 全文过一遍与本章相关的条款；至少明确回答：

- [ ] 主角线是否被稀释？配角是否夺戏？  
- [ ] 境界/时间线/地理是否与 `docs` 追踪表冲突？  
- [ ] 是否触碰「代际主角/接班人」类表述？

结论仅 **通过 / 不通过**。

### 4.2 Soft 门禁（打分）

对方法论 v0.4 的 **S1–S6** 各打 1–5 分 + 一句证据。

- 任一项 **≤2**：本章必须二次修订或回退扩写段落。
- **S1、S2 或 S6 任一 ≤2**：禁止进入下一章扩写，先修本章。
- **基线不退**：扩写后 **S1、S2 均不得低于扩写前记录**（见 §二-5）；若字数已达标但 S1/S2 下滑，**不合格**，须改扩写策略或回滚。

### 4.2.1 修订轮次上限（防死磕）

同一章因 Soft/篇幅门禁打回扩写或润色，**建议不超过 2 轮**；第 3 次仍不达标须 **升级主编会诊**（裁定：接受例外 / 改重写 / 冻结本章），避免单人无限打磨（吸收专家设计 2）。

### 4.3 篇幅门禁

- 字数是否落入团队约定区间？  
- 若仍偏短：说明缺的是 **节拍** 还是 **字数**——缺节拍应加 **冲突/选择** 而非景物。

- **微调分支**（吸收专家设计 2）：字数已在舒适区，但 **S3–S5** 偏低 → **不做强制扩字**，走局部润色（删模板化排比、调对话信息密度、收束节奏），仍须过 Gate 0。

### 4.4 术语与格式

- 势力、技能、地名与 `docs` 或前文 **首现拼写** 一致。  
- 章节标题格式与 `src/volume*/ch*.md` 既有惯例一致。

## 五、交付物（建议）

每章扩写 PR 或批次末尾附：

1. **本章骨架 5 条**（扩写前）  
2. **扩写基线**：扩写前字数 + **S1/S2 分数**  
3. **变更摘要**：加了哪类厚度、未改哪些关键对白/结局  
4. **Hard 通过 + Soft 分数（含扩写后）**；人类抽检报告可采用 `小说评审优化方法论_v0.4.md` **§9.6** 最小记录模板  
5. **需全局跟进的伏笔**（若有，更新 `docs/伏笔追踪.md`）

**扩写试点（POC）批次**须额外产出：**千字扩写有效工时**（或「成稿汉字 / 投入小时」），供 `docs/修订路线图-定案.md` 推算规模化人天（吸收专家设计 3、4）。

---

## 六、回滚

若扩写后 Hard 失败：优先 **回滚扩写段落** 保留原骨架，再拆分为更小批次重写，避免在失败版本上继续堆字。

---

## 七、批次回归与雷区测试集（吸收专家设计 1、4）

### 7.1 每批次合入后（最小集）

- 对本批已改章节，抽查 **相邻 ±3 章** 的 Hard 相关项（是否被新正文带出矛盾）。  
- 若本批含 **P0 / 里程碑 / 曾修复雷区** 章节：将对应章号加入「高优先级复测列表」。

### 7.2 雷区测试集（须维护Living Doc）

建议独立维护 `novel-research/雷区回归测试集.md`（或等价表格），至少包含：

1. **章节子集**：10–15 个关键位（卷首卷末、已知曾违规章、里程碑章）。  
2. **表述子集**：从 `review_prompt.md` 整理的禁忌模式 + **已修复复发案例**；脚本只做**预警**，**语义是否违规由人判**（与方法论 §2.1 一致）。  
3. **境界样例**：白名单句式以 **`docs/境界进度追踪.md` 与本书真实境界名** 为准；**禁止**照抄他书「筑基/金丹」等与本书无关的示例词表（纠偏专家设计 2 类笔误）。

### 7.3 阶段后期抽审

规模化扩写**后期**：从已通过批次中 **随机抽 10%** 章，用最新版 `review_prompt` 再过 Soft，确认未被后续改动间接带崩（吸收专家设计 2）。
