## 二、核心问题与解决方案

### 问题1：字数统计口径不一致
**现象**：agent报告36万字，实际扫描22万字
**原因**：英文字母和标点被计入中文字符统计
**解决**：统一使用`[\u4e00-\u9fff\u3000-\u303f\uff00-\uffef]`正则

### 问题2：模板化重复段落
**现象**：每章末尾有200-400字完全相同的模板段落
**解决**：截断到"**本章完**"标记，删除标记后所有内容

### 问题3：章节编号与文件名错位
**现象**：ch121.md内容实际是第129章
**原因**：扩写时代理错误重命名了文件
**解决**：以文件内容为准，不重命名；建立映射表记录

### 问题4："本章完"截断错误
**现象**：部分章节在"本章完"后截断，但模板文字被截留在正文中
**解决**：先截断到"**本章完**"，再处理后续内容

### 问题5：扩写后章节仍然不达标
**现象**：扩写后仍有章节<2000字
**原因**：扩写幅度不够，或删水后反而减少
**解决**：多轮迭代，每次扩写最短章节直到达标

## 三、扩写方法论（实测有效）

### 6要素扩写法
1. **场景具象化**：五感描写（视觉/听觉/触觉/嗅觉/味觉）
2. **心理活动**：角色内心独白、情绪波动
3. **对话丰富化**：增加潜台词、语气词
4. **动作细节**：肢体语言、微表情
5. **环境描写**：场景氛围、时代背景
6. **章末钩子**：悬念或情节推进

### 批量处理脚本
```python
import re, glob

def count_chinese(text):
    return len(re.findall(r'[\u4e00-\u9fff\u3000-\u303f\uff00-\uffef]', text))

# 1. 扫描所有章节
for f in sorted(glob.glob('src/volume1/ch*.md')):
    with open(f) as fp:
        content = fp.read()
        if '**本章完**' in content:
            content = content.split('**本章完**')[0]
        cnt = count_chinese(content)
        print(f'{f}: {cnt}字')

# 2. 清理重复段落
repetitive_patterns = [
    '见证着这一刻',
    '这就是守护者的',
    '传奇的篇章正在书写',
]
for pattern in repetitive_patterns:
    content = re.sub(pattern, '', content)

# 3. 截断到本章完
if '**本章完**' in content:
    content = content.split('**本章完**')[0] + '**本章完**\n'
```

## 七、教训总结

1. **以实际扫描为准**：agent报告的字数可能是错的（包括padding），必须用Python脚本实际统计

2. **先截断后扩写**：清理"本章完"后残留内容应该在扩写之前，避免重复清理

3. **不要轻易重命名文件**：章节编号与文件名错位后很难修复，应该以文件内容为准

4. **并行化要留补差轮次**：多轮并行扩写后仍会有遗漏，需要专门补差轮次处理最短章节

5. **套话清理是持续工作**：模板化段落会在多轮扩写中反复出现，每轮都要清理

6. **Hard Check优先**：先保证主线主角、境界名等硬约束，再做扩写

## 十二、核心教训

1. **字数必须实际扫描**：agent报告不可信，必须用`[\u4e00-\u9fff]`正则实际统计

2. **并行化≠并行启动**：人类判断会串行化，必须一次发满10路

3. **精修会删字**：每轮精修删减可达1-2万字，预留补差预算

4. **样板句是批量问题**：建立替换库，一次性替换52+处

5. **读者评审要找对人**：10种不同背景的读者才能覆盖所有问题

6. **P0优先处理**：战力崩溃、逻辑漏洞等必须在扩写前修复

7. **修复要有记录**：每次修复后更新文档，形成知识沉淀
