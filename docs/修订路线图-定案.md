## 一、北极星与门禁顺序（已定默认）

**执行顺序**（每章 / 每批次合入前）：

1. **Gate 0 — Hard**：仅「通过 / 不通过」；不通过则回滚或走重写分支。  
2. **Gate 1 — 篇幅**：第二、三卷目标 **2000–3500 字/章**（统计口径：**正文汉字 + 全角标点**，不含章标题、不含仅分隔用 `---`、不含纯空行）。高潮章超上限须 **主编特批** 并记入 `HISTORY.md`。 **【主编可改：区间上下限】**  
3. **Gate 2 — Soft**：S1–S5 打分见 `../novel-research/小说评审优化方法论_v0.4.md` §2.3；**扩写基线不退**见同文档 §2.4；批次要求：**均分 ≥3.5** 且 **不得有两项同时 ≤2**；任一章 S1 或 S2 **≤2** 则该章禁止进入下一批次扩写链，须先修。**人审抽样与缺陷分级**默认见同文档 **第九章「评审作业规范」之 9.3、9.4**（可书面调整）。

**不采用**「每 500 字必须 N 个冲突点」作为合入硬指标；可作为扩写执行**自检参考**（见 SOP §3.3）。

## 二、战术路径（扩写 / 重写 / 删水）

### 2.1 判定（与方法论 §6 一致）

| 情形 | 动作 |
|------|------|
| Hard 违规，或里程碑与追踪表冲突且扩写无法修 | **重写**（必要时回溯依赖章） |
| 因果链断、动机不成立 | **局部重写** |
| 骨架正确、篇幅不足或缺场景与情绪 | **扩写**（SOP 五类加厚） |
| 字数在 Gate 1 内但 S3–S5 偏低 | **润色**，不强制扩字数 |
| v1 超长且信息重复 | **删水**（见 §2.3） |

**重写占比预警**：与 `../novel-research/小说评审优化方法论_v0.4.md` **§2.5** 一致——基线评审后，若预估重写章 **> 待处理子集 20%**，暂停规模化扩写，**主编会议**评估是否调整大纲或切块重写。 **【主编可改：20% 阈值】**

### 2.2 第二、三卷（已定默认）

- 以 **分阶段扩写** 为主，**局部重写**为辅。  
- 扩写前必读 **相邻章 ±1**；P0 / 卷间衔接章建议 **±3**。

### 2.3 第一卷（已定默认）

| 项目 | 定案 |
|------|------|
| 默认策略 | **不主动压篇幅**；v1 作为密度与风格样本。 |
| 删水触发 | 单章 **>4500 字** 且评审认定 **重复信息 / 同义反复 / 无效寒暄** 占比高（具体比例 **【主编可改：建议 ≥20%】** 由首轮基线后书面定）。 |
| 删水下限 | 删水后该章 **不低于 2500 字**（若与商业方合同冲突，以合同为准并改本文）。 **【主编可改：2000–3000 区间任选写死】** |
| 禁止 | 删除唯一因果链、里程碑对白、`docs/伏笔追踪.md` 已登记锚点（除非同步废弃伏笔）。 |

### 3.1 总览

| 阶段 | 目标 | 主要输出 | 验收要点 |
|------|------|-----------|----------|
| **0 基线评审** | 360 章（或二三卷子集）Hard/Soft 与篇幅地图 | 分级清单（P0–P3）、扩写/重写/删水标注 | P0 章号与 `review_prompt`、PROJECT_PLAN P0 **对齐**；统计口径团队确认 |
| **1 P0 修复** | Hard 与结构性硬伤清零 | 修订章节、更新 `docs/` 三追踪表、`HISTORY` 记录 | Gate 0 全通过；不引入新 S1/S2≤2 |
| **2 扩写试点** | 校验 SOP 与工时 | **5 章**（建议 v2 2 + v3 2 + **v1 对照 1** 仅风格对齐不强制扩） | Gate 1–2 达标；形成「样板章」与**千字人时** |
| **3 规模化** | 按 P1→P2→P3 批量扩写/重写/删水 | 分批 PR/批次报告、追踪表增量更新 | 每批 Gate 0 零事故；Gate 1 **达标率 ≥90%**（余量特批）；Soft 抽样达标 |

### 3.2 人天与编制（待试点校准）

| 角色 | 最少配置 | 职责摘要 |
|------|----------|----------|
| 主编 | 1 | 裁示、终验、特批篇幅、冻结范围 |

## 四、质量门禁与回归（已定默认）

### 4.1 合入前检查单（最小集）

- [ ] Gate 0：`review_prompt` 相关条 + 三追踪表无冲突  
- [ ] Gate 1：字数在约定区间或已特批  
- [ ] Gate 2：Soft 满足 **本文 §一** 所列阈值（与 `../novel-research/小说评审优化方法论_v0.4.md` §2.3、§2.4 及第九章抽样一致）  
- [ ] 相邻章读毕证明（执行人在 PR 说明中列「已读 chXXX±1」）  
- [ ] 若动伏笔/境界：`docs` 已同步

### 4.2 回归策略

| 类型 | 做法 |
|------|------|
| 增量 | 每章改动后跑 **禁忌关键词脚本**（预警，非自动定罪） |
| 批次 | 本批章节 **±3 章** Hard 复核；P0 修复后抽 **同类型里程碑章** |
| 终盘 | 全卷 Hard；Soft **随机 20%**；卷间衔接 **人工通读** ch120/121、ch240/241 等 |

**境界回归**：白名单 / 样例句式以 **`docs/境界进度追踪.md` 与本书实际境界名** 为准（**禁止**沿用与本作无关的「筑基/金丹」等外书示例）。

### 4.3 终验「方差」（已定默认）

- **不采用**「三卷单章字数方差 ≤15%」作为唯一硬终验。  
- **采用**：① Gate 1 **达标率 ≥90%**（二三卷主体）；② 分卷字数**中位数**差距在基线表上**可追溯、可解释**；③ 主编对卷末/卷首 **抽读通过**。 **【主编可改】**
