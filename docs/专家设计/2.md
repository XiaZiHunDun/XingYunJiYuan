# 《星陨纪元》修订与质量提升 · 专家方案
## 1. 结论摘要（10 条以内要点）
1. **北极星指标**采用「Hard 风险清零为前置门槛 + 单章有效字数区间（2000-3500字）为主 + Soft S1/S2 均分≥3.5」的混合驱动。
2. **战略取舍**：优先保「控制总工期」与「二三卷体验统一」，适度牺牲「第一卷现有密度的绝对保留」（通过冻结 v1、以 v1 为标尺映射 v2/v3 来补偿体验）。
3. **第一卷策略**：**绝对不动**。严禁为拉齐方差而压缩 v1，仅对极个别超 4500 字且明显注水的章节做微观删水。
4. **第二、三卷策略**：以**分阶段扩展**为绝对主力（预估占 85%），仅对因果断裂或里程碑缺失的章节触发**局部重写**（占 15% 内）。
5. **判定树核心**：是否违反 Hard 约束？是→重写/回溯；否→骨架是否成立？成立→扩写；不成立→重写。
6. **工期预估**：总计约 85-95 人天，按 3 人核心团队并行，可在 4-4.5 个月内完成全量修订与统稿。
7. **分工底线**：最少需 3 个明确角色（设定监理/扩写执行/终稿评审），设定监理必须由对全书设定最熟的人担任，且拥有 Hard 一票否决权。
8. **防注水机制**：严格执行 SOP 中的「每 800 字自检」与「五类加厚禁止项」，拒绝无人物反应的景物堆砌与无后果动作戏。
9. **回归测试**：建立「红线词库」（如接班人、下一代主角等）与「里程碑快照」，每次合入前必须通过自动化脚本扫描+人工抽检 10%。
10. **商业默认假设**：按「网文平台连载向」默认，强调章末钩子与单章信息密度；若后期转出版，再通过统稿阶段合并过渡章。

## 2. 战略取舍与北极星指标
### 2.1 优先级排序与取舍
1. **控制总工期**（P0）：项目必须有可度量的闭环，避免陷入无限打磨。
2. **三卷阅读体验统一**（P1）：解决二三卷因压缩导致的「骨架感」和「薄脆感」。
3. **保留第一卷现有密度**（P2）：砍掉此项中的「绝对不妥协」。
**补偿策略**：不压缩 v1，而是将 v1 视为「风格与密度锚点」。在二三卷扩写时，扩写执行人必须对照 v1 同类型场景（如日常、战斗、突破）的字数分布与描写比例，做「映射式扩写」，而非无序拉长。
### 2.2 北极星指标设计（混合型）
- **Gate 0（一票否决）**：Hard 门禁 100% 通过（主角唯一性、里程碑、禁令词）。
- **Gate 1（主指标，权重 60%）**：单章有效字数落入 2000–3500 字区间。统计口径固定为「正文汉字+全角标点」，排除标题与空行。
- **Gate 2（辅指标，权重 40%）**：Soft S1（剧情完整性）与 S2（逻辑自洽）均不低于 3 分，且单批次抽样均分 ≥3.5。

## 3. 战术路径（扩写 / 重写 / 混合的判定树）
所有进入修订池的章节，按以下流程分流：
```text
起始：章节进入修订池
  │
  ├─ 触发 Hard 违规（主角夺权/里程碑错位/因果断裂）？
  │    ├─ 是 → 【重写分支】标为 P0，拉取前后 3 章对表，回溯修订依赖链。
  │    └─ 否 ↓
  │
  ├─ 骨架事件顺序正确，但字数 < 1500 或节拍不足？
  │    ├─ 是 → 【扩写分支】进入章节扩展 SOP 流程（对表→五类加厚→800字自检→门禁）。
  │    └─ 否 ↓
  │
  ├─ 视角漂移至非林夜/苏琳且无主线服务价值？
  │    ├─ 是 → 【删改分支】先删旁支/改视角，再评估是否需走【扩写分支】补回字数。
  │    └─ 否 ↓
  │
  └─ 字数在区间内，但 Soft 得分低（如 S3 文笔差/S5 节奏平）？
       └─ 【微调分支】不触发扩写/重写，走局部润色（替换模板化排比、调整对话信息密度）。
```
**第一卷（v1）专属路径**：
- 默认动作：**跳过**，直接标记为「基线冻结」。
- 唯一触发条件：单章字数 > 4500 字 **且** 人工评审明确标注「同义反复/无效寒暄」。
- 处理方式：仅做「删水型压缩」，**严禁删除核心冲突与因果链**，压缩后不得低于 3000 字。

## 5. 组织与分工（RACI）
| 角色 | 最小配置 | 职责定义 | R (执行) | A (负责) | C (咨询) | I (知会) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **设定监理** | 1人（必须是设定最熟者） | 对照 `docs/` 三张追踪表，守 Hard 门禁，管里程碑与时间线。 | 硬伤核查 | Hard否决权 | - | 全流程 |
| **扩写执行** | 1-2人（人或人机协作） | 按 SOP 执行加厚，做 800 字自检，保证字数区间。 | 扩写起草 | - | 监理/主编 | - |
| **主编/统稿** | 1人 | 把控 Soft 分数（S3-S5），判定风格一致性，处理批次间衔接。 | Soft评审/统稿 | 最终交付质量 | 监理 | - |
| **终稿评审** | 可由主编兼 | 阶段五全盘审视，签发入库。 | 终审 | 项目闭环 | 监理 | 项目干系人 |
**人机协同边界**：
- **机器做**：字数统计、红线词正则匹配、基于 SOP 的初步扩写草案生成。
- **人必须做**：Hard 约束判定（语义层面，非字面匹配）、Soft S1/S2 打分、五类加厚中的「情绪波动」与「决策代价」写入、终稿情感共鸣校验。

## 6. 质量门禁与回归策略
### 6.1 门禁矩阵（对齐方法论 v0.2）
- **前置门禁（合入前）**：100% 跑通 Hard 检查清单（对照 `评审清单与项目文档对照表.md` 第一节）。
- **批次门禁（每 10-15 章）**：
  - S1、S2 无 ≤2 分项。
  - S1-S5 抽样均分 ≥3.5。
  - 字数区间命中率 ≥90%（允许少量高潮章超 3500 字或极短过渡章略低，但需备注）。
- **发布门禁（阶段五）**：全书红线词 0 命中，卷间衔接（ch120/121, ch240/241 等）人工通读无卡顿。
### 6.2 回归测试策略（防复发）
1. **静态词表巡检**：提取 `review_prompt.md` 中的禁忌表述（如「下一代」「接班」「传承大统」等），使用脚本对 `src/volume*/` 每次变更后做 diff 级扫描。
2. **里程碑快照锁死**：境界突破章的描述词（如「筑基」「金丹」及对应异象），在 `docs/境界进度追踪.md` 中设定为正则白名单，非白名单表述触发预警。
3. **10% 随机抽审回溯**：在阶段四后期，从已通过的阶段二、三中随机抽 10% 章节，用最新版 prompt 重新跑 Soft 评审，确认未因后续修改引入新矛盾。

## 7. 风险与缓释
| 风险 | 等级 | 缓释措施 |
| :--- | :--- | :--- |
| **1. 扩写导致「AI感」或模板化注水**（如环境描写与人物反应脱节） | 高 | 严格执行 SOP §3.1「禁止」列与 §3.3「自检」机制；设定监理在抽审时拥有对「无人物反应的景物」直接打回的权力。 |
| **2. 视角在扩写中无意漂移至配角/下一代** | 高 | 将「代际主角」红线词库升级为语义级巡查（不仅查词，查「主语是否从林夜/苏琳变更为徒弟/后辈超过连续两段」）。 |
| **3. 工期蔓延（无限打磨 S3/S4/S5）** | 中 | 锁定「S1/S2 为高优，S3-S5 为中优」原则；设立单章修改上限（最多打回重扩 2 次，第 3 次升级至主编会诊，避免单人死磕）。 |
