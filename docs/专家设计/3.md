# 《星陨纪元》修订与质量提升 · 专家方案

## 1. 结论摘要（10 条以内要点）

1. **采用「分阶段扩展 v2/v3」方案（A方案）**：以第一卷为密度参考，不强制压缩第一卷，专注恢复第二、三卷厚度。
2. **北极星指标**：单章有效字数区间（2000–3500字）+ S1/S2 软指标均分≥3.5，双轨并行。
3. **Hard 门禁一票否决**：主角唯一性、里程碑、卷衔接、禁止代际 handoff 表述，任何违反立即回滚。
4. **判定树驱动**：每章先过「扩写 vs 重写」判定树，避免在断裂骨架上堆字数。
5. **试点先行**：先完成5章扩写试点（v2选2章+v3选2章+v1对照1章），验证SOP与工时估算。
6. **人机协同**：模型负责初筛（字数统计、术语一致性扫描），人类负责Hard终审与情感曲线判断。
7. **版本冻结机制**：每批次扩写前冻结基线，扩写后必须通过Hard+Soft双门禁方可合入。
8. **回归测试清单**：针对已修复的禁忌表述、境界描述建立自动化扫描脚本，防止复发。
9. **工期估算**：全书360章，预计4–6个月完成，分3个sprint，每sprint交付1卷当量。
10. **受众默认假设**：网文平台连载读者（节奏敏感），但保留出版适配空间（单章可独立成篇）。

## 2. 战略取舍与北极星指标

### 2.1 三要素优先级排序

| 优先级 | 要素 | 取舍理由 |
|--------|------|----------|
| **P0** | 控制总工期 | 6个月是团队心理与资金双重红线，超期则项目风险指数级上升 |
| **P1** | 三卷阅读体验统一 | 以v1为锚，v2/v3向v1对齐，而非双向压缩，降低复杂度 |
| **P2（可砍）** | 保留第一卷现有密度 | 若试点后发现v1确有「信息重复」硬伤，可局部删水，但不主动压缩 |

**补偿读者体验**：若工期紧张，优先保证卷末高潮章（ch120、ch240、ch360）与卷首衔接章的质量统一，过渡章允许方差。

### 2.2 北极星指标定义

采用**混合指标**，权重如下：

| 指标 | 权重 | 说明 |
|------|------|------|
| 单章字数区间 | 40% | 2000–3500字为绿区，<1500字红区需扩写，>4500字黄区评估删水 |
| S1（剧情完整性）+ S2（逻辑自洽） | 35% | Hard 通过后评分，任一项≤2则禁止进入下一章 |
| S3–S5（文笔/情感/节奏） | 25% | 批次均分≥3.5，用于风格一致性监控 |

**不采用**「卷内节拍/冲突密度」作为主指标——因该指标主观性强，且与字数高度相关，易重复计算。

## 3. 战术路径（扩写 / 重写 / 混合的判定树）

```
每章入口
  │
  ├─ 是否触碰 Hard 约束？（主角唯一性/里程碑/卷衔接/代际表述）
  │   ├─ 是 → 标记 P0，进入「重写」分支，暂停扩写
  │   └─ 否 → 继续
  │
  ├─ 骨架事件顺序是否正确？（目标-障碍-结果是否成立）
  │   ├─ 否 → 标记「重写」，回溯修订依赖章
  │   └─ 是 → 继续
  │
  ├─ 字数 < 1500字？
  │   ├─ 是 → 进入「扩写」分支（五类加厚：环境/心理/对话/动作/背景）
  │   └─ 否 → 继续
  │
  ├─ 字数 > 4500字 且 评审认定信息重复？
  │   ├─ 是 → 进入「删水压缩」分支
  │   └─ 否 → 维持现状，仅Soft评分归档
  │
  └─ 结束
```

**判定标准细化**：

| 情况 | 处理方式 | 触发条件 |
|------|----------|----------|
| **扩写** | 五类加厚，不改变因果与结局 | 骨架正确，字数<1500或节拍明显不足 |
| **重写** | 拆解骨架，重排因果链 | 里程碑无法满足、整段因果断裂、Hard约束冲突 |
| **改视角/删旁支** | 先修正视角漂移，再评估扩写 | 配角夺戏但未破坏Hard，可快速修复 |
| **删水压缩** | 删重复信息、同义反复、无效寒暄 | 字数>4500且重复率高，或同一比喻结构连续两次以上 |

## 6. 质量门禁与回归策略

### 6.1 Hard门禁（一票否决）

| 检查项 | 方法 | 责任人 |
|--------|------|--------|
| 主角线是否被稀释？配角是否夺戏？ | 对照`review_prompt.md`一、三节，全文检索「接班人」「传承给」「下一代主角」等关键词 | 设定监理 |
| 境界/时间线/地理是否与追踪表冲突？ | 逐章比对`境界进度追踪.md` | 设定监理 |
| 是否触碰「代际主角/接班人」类表述？ | 正则扫描+人工语境判断 | 设定监理 |
| 卷间关键衔接（ch120/241等）是否成立？ | 精读衔接章+下一章开头 | 项目主编 |

**结论**：仅「通过 / 不通过」，不通过立即回滚扩写段落。

### 6.2 Soft门禁（评分驱动）

| 维度 | 阈值 | 动作 |
|------|------|------|
| S1（剧情完整性） | ≤2 | 禁止进入下一章，优先修复 |
| S2（逻辑自洽） | ≤2 | 禁止进入下一章，优先修复 |
| S3–S5 | ≤2 | 批次内二次修订或回退 |
| 批次均分 | <3.5 | 暂停扩写，先修风格指南或模板问题 |

### 6.3 回归测试策略

**问题**：已修复的禁忌表述、境界描述在后续扩写中复发。

**措施**：

1. **建立「已修复问题清单」**：记录每处P0修复的章节、原文、修改后文本、触发原因。
2. **自动化扫描脚本**（建议）：
   - 输入：新扩写批次文本
   - 扫描：已修复问题关键词、境界描述模板、代际表述模式
   - 输出：疑似复发标记+置信度
3. **人工复核**：设定监理对高置信度标记进行语境判断，确认复发则回滚并更新脚本规则。
4. **版本冻结**：每批次扩写前冻结基线，扩写后必须通过回归测试方可合入主线。

## 7. 风险与缓释

| 风险 | 概率 | 影响 | 缓释措施 |
|------|------|------|----------|
| **试点估算偏差，导致总工期失控** | 中 | 高 | Sprint 1强制产出「工时/千字」数据，若偏差>30%则调整Sprint 2–3批次大小或增配人力 |
| **v2/v3骨架大面积断裂，扩写无法救** | 中 | 高 | Sprint 0基线评审强制标记「扩写/重写」判定，若重写章>20%则触发方案重评（是否切换D方案：局部重写为主） |
| **Hard约束在规模化阶段复发** | 中 | 极高 | 回归测试脚本+设定监理抽检，每批次抽10%复测已修复问题 |
| **风格统一失败，三卷仍像多人写作** | 低 | 中 | Sprint 3强制产出风格指南v1.0，统稿阶段执行「术语+语气」对齐，终稿评审加入风格一致性抽样 |
| **团队疲劳导致Soft指标放水** | 中 | 中 | 终稿评审由外部或轮换人员执行，Soft打分与扩写执行人员隔离 |
