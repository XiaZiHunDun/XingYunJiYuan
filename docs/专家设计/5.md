# 《星陨纪元》修订与质量提升 · 专家方案

## 1. 结论摘要（10 条以内要点）

1.  **战略优先级排序**：**风险清零 > 体验统一 > 控制总工期**。牺牲工期不可接受，但牺牲前两者会动摇IP根基。
2.  **北极星指标**：采用 **混合指标**，以 **“Soft均分（S1+S2）≥ 3.5”为首要KPI**，辅以“单章字数1500-4500字”为舒适区边界。
3.  **默认战术**：对第二、三卷采用 **“分阶段扩展”为主**。仅当因果链断裂、里程碑无法满足时，才启动 **“局部重写”**。
4.  **第一卷策略**：**原则上不主动压缩**，仅做“删水”处理。将其作为全书的“密度基准”和“风格样本”。
5.  **核心流程**：建立 **“基线评审 → P0修复 → 扩展试点 → 规模化扩展 → 全书统稿”** 五阶段路线图。
6.  **最小组织配置**：至少需要 **1名主编**（负责战略与终审）、**1名设定监理**（负责`docs/`对表）、**2名扩写执行**（人机协同）、**1名统稿**（负责润色与格式）。
7.  **关键门禁**：**Hard门禁** 作为拦马索，任何改动不得违反；**Soft量表（S1/S2）** 作为进度控制阀，均分不达标则暂停批次。
8.  **回归策略**：建立 **“核心设定与禁令测试集”** ，在每个扩写批次后对相关章节进行自动化/半自动化回归抽查，防止禁忌表述复发。
9.  **最大风险**：**扩写导致注水、主角线被稀释**。缓释措施是严格执行SOP中的“五类加厚”与“每800字自检”。
10. **默认受众假设**：在方案中默认以 **“追求质量的网文核心读者”** 为目标，兼顾出版单行本的可读性。节奏上强调“章末钩子”与“卷内高潮点”。

## 2. 战略取舍与北极星指标

### 2.1 战略优先级排序
1.  **风险清零（P0）**：确保所有`review_prompt.md`中的Hard约束（主角唯一性、里程碑、卷间衔接）零事故。这是IP的底线。
2.  **体验统一（P1）**：消除“第二、三卷过瘦”带来的仓促感，使三卷阅读体验平滑，信息密度接近。
3.  **控制总工期（P2）**：在保证前两项的前提下，通过分阶段、试点先行、工具辅助等方式控制人力成本。

**取舍与补偿**：如果必须保两项，我选择保 **风险清零 + 体验统一**，适度延长工期。**补偿策略**：将“控制总工期”的压力转化为 **“流程提效”**——通过更精准的批次划分、更严格的门禁减少无效返工，总体人天反而可能低于无章法的全面重写。

### 2.2 北极星指标
建议采用 **混合指标**，明确权重：

- **首要KPI（60%权重）**：**“高风险章节（P0/P1）及抽样章节的Soft均分（S1剧情完整性 + S2逻辑自洽）≥ 3.5分”**。这直接指向“厚度与质量”。
- **次要KPI（30%权重）**：**全卷单章字数中位数落入 [1800, 4000] 字区间**。比舒适区（2000-3500）略放宽，尊重高潮章的特殊性。
- **否决性指标（10%权重）**：**Hard门禁100%通过**。任何一章违反，该批次直接打回。

**决策理由**：字数只是表象，信息量与阅读体验才是本质。先通过Soft指标确保章节“有料”，字数自然会趋向合理区间。避免为了凑字数而注水。

## 3. 战术路径（扩写 / 重写 / 混合的判定树）

我将您的“骨架”概念具体化为以下**判定树**，供扩写执行人使用：

**第一步：Hard约束检查（前置门禁）**
- 本章是否涉及`review_prompt.md`中的【卷间关键衔接】或【修炼里程碑】？
    - **是** → 必须与`境界进度追踪.md`、`PROJECT_PLAN.md`严格对表。若有冲突，**进入【重写分支】**。
    - **否** → 进入下一步。

**第二步：因果链与骨架评估**
- 本章的事件顺序、结果、关键对白是否被采纳为定稿（无需改变故事走向）？
    - **否**（例如：主角动机断裂、关键道具凭空出现）→ **进入【重写分支】**（至少重写该段落）。
    - **是** → 进入下一步。

**第三步：厚度与节拍评估（Soft初评）**
- 依据方法论v0.2的S1/S2快速自评：
    - 本章节拍是否过密（几个关键事件挤在一起无喘息）？/ 是否缺乏场景、心理、环境细节？
        - **是** → **进入【扩写分支】**。执行`章节扩展标准操作手册.md`，补环境、心理、对话、动作、背景。
    - 章节字数是否<1500字，且节拍明显缺失？
        - **是** → 同上，**进入【扩写分支】**，优先补充缺失的节拍。
    - 章节字数>4500字，且存在明显重复信息？
        - **是** → **进入【压缩分支】**，按§5.2删水，保留核心因果链。

**总结**：默认走**扩写**，因果断裂走**重写**，第一卷超长章走**选择性压缩**。

## 6. 质量门禁与回归策略（对齐我方 Hard/Soft）

### 6.1 质量门禁（扩写批次合入前必须通过）

1.  **Hard门禁（拦马索）**：执行`章节扩展标准操作手册.md` §4.1。必须全绿。
2.  **Soft量表门禁（进度阀）**：
    - 本批次所有章节 **S1与S2单项均分 ≥ 3.5**。
    - 本批次**无任何一章**的S1或S2得分 ≤ 2。
    - 若违反，主编裁定是“修订本批次”还是“暂停并回溯SOP”。
3.  **篇幅门禁**：本批次章节字数100%落入[1500, 5000]区间（更宽的闸门，避免误杀高潮章）。超上限需主编特批并说明理由（如大战场）。
4.  **术语与格式门禁**：与`docs/`及前文一致，通过`统稿`检查。

### 6.2 回归策略（防止复发）

建立 **《核心设定与禁令测试集》**，示例条目：
- **主角唯一性**：检查是否出现“新一代”、“继承一切”、“故事交给...”等模式化短语。
- **境界里程碑**：检查林夜/苏琳在`境界进度追踪.md`记录的突破章节，描述是否一致。
- **卷间衔接**：检查ch120结尾与ch121开头、ch240与ch241等，是否存在时间/地点跳跃导致的断裂。
- **代际传承**：检查是否出现“让下一代成为主角”类表述。

**执行方式**：
1.  **自动化初筛**：编写简单脚本，对每批次的增量文本，检索测试集中的**关键词/正则模式**（如`继承.*一切|交给.*下一代`）。命中则预警。
2.  **半自动化复核**：主编或设定监理对预警进行人工确认。
3.  **人工抽审**：每批次随机抽取20%章节，对照测试集进行人工逻辑检查。
4.  **迭代更新**：每次发现新的复发模式，立即加入测试集。
